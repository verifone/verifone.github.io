Detailed Description

This represents the data that is shared between the terminal and the Android application.

Setting the fields is only relevant when sending the data to the terminal, and conversely, getting the fields is only relevant when receiving this object from the terminal.

abstract String getAuthCode ( )
abstract

The transaction Authorization Code usually provided by the payment processing service or acquirer.

This may be added during the transaction, but once received will not be changed. This may not be set.

abstract String getAuthResponseText ( )
abstract

Descriptive text accompanying the authorization code from payment processor.

Usually present when the transaction has failed authorization this field provides the reason for the failure. This may not be set.

abstract String getBin ( )
abstract

The first six digits of the card holder PAN.

Generally not present alongside the last four digits, this is only provided during specific triggers.

abstract String getPanHandle ( )
abstract

A reference returned by the Payment Application to refer to the PAN in question.

Commerce Application may choose to use PAN_Handle to request an encrypted PAN via API CP_APP_REQUESTS_ENCRYPTED_CARD.
