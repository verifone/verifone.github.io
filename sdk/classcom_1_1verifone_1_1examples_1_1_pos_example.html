Detailed Description

Example POS app for the Commerce Platform Payment API.

In a typical flow, the session and basket will be opened, reserving the payment terminal to be able to display items on screen as they are added to the basket. The cashier will add items to the basket using the screen or a scanner, calling the addMerchandiseToBasket(Merchandise merch) method, which adds the item to the basket and updates the display with the line item and the updated running total. Once the cashier has finished adding items, the basketFinished(Merchandise merch) should be called, allowing the basket to be read and updated by registered applications. Once the basket is finished, processSale() may be called, starting the transaction for current basket total. As payments are received, the handleEvent(CommerceEvent event) method is called with a

TRANSACTION_PAYMENT_COMPLETED

event, allowing the appropriate receipt to be printed. Finally, the handleEvent(CommerceEvent event) method is called with a

TRANSACTION_ENDED

event, allowing the cashier to begin a new basket and transaction.

Please note this class is built as an example, and therefore favors simplicity and accessibility over Android best practices.
