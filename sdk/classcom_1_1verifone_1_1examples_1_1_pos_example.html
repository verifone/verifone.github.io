Detailed Description

Example POS app for the Commerce Platform Payment API.

This class is built as an example and is not compilable source code, but is meant to provide direction. It therefore favors simplicity and accessibility over Android best practices.

In a typical payment flow, the steps are as follows:

  • A transaction session is opened, reserving the payment terminal for use by this app.
    mTransactionManager.startSession(this);
  • In the Commerce Listener, wait for the Session Started or Session Resumed event before continuing. A session must be open to be able to do a transaction, or update the basket on the customer-facing screen, or perform other similar actions.
    public CommerceEvent.Response handleEvent(CommerceEvent event)
    CommerceEvent.SESSION_STARTED
    CommerceEvent.SESSION_RESUMED
  • The basket is opened, allowing the app to add merchandise to the basket and update the customer-facing screen with the items and running total.
    mBasketManager = mTransactionManager.getBasketManager();
  • As items are entered added in the POS app, the items should be added to the basket which triggers the customer-facing display to update with the new item and the new running total.
    mBasketManager.addMerchandise(merch);
  • Once all of the items have been added, we finalize the basket, allowing any other apps to perform any necessary modifications.
    mBasketManager.finalizeBasket();
  • Once the basket is finalized, we start the payment process.
    AmountTotals currentTotals = mBasketManager.getCurrentAmountTotals();
    Payment payment = new Payment();
    payment.setRequestedPaymentAmount(currentTotals.getRunningTotal();
    mTransactionManager.startPayment(payment);
  • As payments are received, the handleEvent(CommerceEvent event) method is called with a PaymentCompletedEvent object, allowing the appropriate receipt to be printed.
    PaymentCompletedEvent payEvent = (PaymentCompletedEvent) event;
    payEvent.getPayment().getReceipt().print();
  • Finally, we end the session, completing the transaction.
    mTransactionManager.endSession();

This description only covers a typical simple payment flow. More complicated flows can be triggered at various points in this process, but are described in more detail in the technical documentation.
